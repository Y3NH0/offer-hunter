一、 AI Mentor (50%)：提升系統使用彈性、可擴展性、實際應用準確度

專注於優化 AIM 的底層核心功能、支援開發維運，並透過數據分析持續提升應用準確度。

1. 底層核心功能開發與優化 ➡️ 提升系統使用彈性、可擴展性

LLM 介面重構

統一用 LangChain ChatModel 串接 LiteLLM，簡化修改成本。
提供 run_llm 功能以利後續串接 tracing 等功能。

3-Level NodeConfigProvider

實現精準 LLM、Retriever、Prompt 控制，以提高客製化彈性。

為 BaseAgent NodeConfigProvider 奠定基礎。

General Collection (RAG Example)

建立 General Example 管理及匯出匯入機制，同時讓 Example 可支援熱改動。

Emotion API 整合

整併進入 Agent Pod，大幅提升後續優化效率。

2. 自動化課綱生成 ➡️ 降低團隊維運成本、提供Playground基礎

End-2-End 上傳檔案後直接獲得生成結果，降低維運時間成本。

二、 直播帶貨 AI 客服小編 (30%)：設計核心功能 API/Graphs/Nodes

負責設計開發此專案的核心 AI Workflow，成功交付應用場景，協助合作廠商實現自動化。

顧客私訊分析

  • 痛點： 處理客戶多重意圖交織、上下文追蹤困難的問題。
  • 設計： 將任務拆解為多個 LLM 子任務 (如意圖分類、未回覆訊息識別)，並導入 Reasoning 輸出以提升準確性。
  • 成果： 準確提取並分類未回覆意圖，並自動爬取訂單、庫存等詳細資訊。

直播留言分析

  • 痛點： 應對大量快速的即時留言，並滿足不同直播主的客製化需求。
  • 設計： 利用 LLM 進行即時訊息分類與摘要，並提供「動態配置類別」功能。
  • 成果： 讓直播主或小幫手能輕鬆掌握聊天室重點，即時回答關鍵問題。
